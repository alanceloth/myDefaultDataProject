Descrição do Projeto: Dieta Econômica - Dashboard Interativo de Preços de Alimentos

Visão Geral

O projeto Dieta Econômica objetiva fornecer aos usuários um dashboard interativo que apresenta uma análise abrangente dos preços históricos e atuais dos produtos alimentícios essenciais para suas dietas. Baseando-se em dados extraídos diretamente do Ifood através de tecnologia de webscraping, o projeto oferece insights valiosos sobre variações de preços, identifica produtos em promoção ou com preços acima da média, e permite aos usuários tomar decisões informadas para manter uma dieta equilibrada enquanto economizam.

Arquitetura e Tecnologias

O projeto emprega um pipeline robusto de dados compreendendo as seguintes etapas e tecnologias:

Extração

  • Apache Airflow: Utilizado para orquestrar e agendar tarefas de extração de dados de forma automatizada.
  • Webscraping (Requests + BeautifulSoup): Técnica empregada para capturar dados pertinentes diretamente das páginas HTML do Ifood, incluindo detalhes de produtos como preços e marcas.
  • Armazenamento RAW (DuckDB): Os dados extraídos são armazenados na forma bruta (RAW) no DuckDB, um banco de dados orientado a colunas, otimizado para consultas analíticas.

Transformação

  • Pydantic: Utilizado para realizar a validação de schemas, garantindo a integridade e correção dos dados em diferentes estágios do processo.
  • DBT (Data Build Tool): Ferramenta fundamental na etapa de transformação para modelar, testar e limpar os dados, preparando-os para análises futuras.
  • Camadas de dados: Dados brutos (Bronze) são transformados para um estado limpo e estruturado (Silver), facilitando análises e visualizações.

Carregamento e Visualização

  • Airbyte: Para a movimentação segura dos dados transformados para um banco de dados robusto, como o PostgreSQL, onde os dados são armazenados na forma Gold, prontos para análise.
  • PostgreSQL: Banco de dados relacional para armazenamento final, habilitando operações de consulta eficientes e complexas.
  • Streamlit: Framework utilizado para desenvolver o dashboard interativo, proporcionando uma interface amigável para análise e visualização de dados.

Funcionalidades

  • Monitoramento do Preço da Dieta: Avalia o custo total da dieta em tempo real, comparando com a mediana histórica.
  • Identificação de Promoções e Preços Elevados: Destaca produtos com preços abaixo da mediana (promoções) e acima da mediana (preços elevados), auxiliando na escolha de alternativas econômicas.
  • Comparativo de Marcas e Produtos: Apresenta opções de marcas para um mesmo produto, indicando a alternativa mais vantajosa financeiramente.
  • Evolução dos Preços: Traça um histórico da variação de preços dos itens da dieta e do custo total, oferecendo uma perspectiva longitudinal da economia na alimentação.

Conclusão

O projeto Dieta Econômica representa um avanço significativo na forma como os indivíduos podem monitorar e gerenciar os custos de sua alimentação. Combinando uma abordagem sofisticada de extração e análise de dados com uma poderosa interface de visualização, o projeto encoraja decisões informadas, promovendo ao mesmo tempo uma alimentação saudável e econômica.

Project Description: Economical Diet - Interactive Dashboard of Food Prices

Overview

The Economical Diet project aims to provide users with an interactive dashboard that presents a comprehensive analysis of the historical and current prices of essential food products for their diets. Relying on data extracted directly from Ifood through web scraping technology, the project offers valuable insights into price variations, identifies products on promotion or with prices above the median, and enables users to make informed decisions to maintain a balanced diet while saving money.

Architecture and Technologies

The project employs a robust data pipeline comprising the following stages and technologies:

Extraction

  • Apache Airflow: Used to orchestrate and schedule automated data extraction tasks.
  • Web Scraping (Requests + BeautifulSoup): The technique used to capture relevant data directly from Ifood's HTML pages, including product details such as prices and brands.
  • RAW Storage (DuckDB): The extracted data are stored in their raw form (RAW) in DuckDB, a column-oriented database optimized for analytical queries.

Transformation

  • Pydantic: Used for schema validation, ensuring data integrity and correctness at different stages of the process.
  • DBT (Data Build Tool): A crucial tool in the transformation stage for modeling, testing, and cleaning data, preparing it for future analyses.
  • Data Layers: Raw data (Bronze) is transformed into a clean and structured state (Silver), facilitating analyses and visualizations.

Loading and Visualization

  • Airbyte: For the secure movement of transformed data to a robust database like PostgreSQL, where data is stored in Gold form, ready for analysis.
  • PostgreSQL: A relational database for final storage, enabling efficient and complex query operations.
  • Streamlit: The framework used to develop the interactive dashboard, providing a user-friendly interface for data analysis and visualization.

Features

  • Diet Price Monitoring: Evaluates the total cost of the diet in real-time, comparing it with the historical median.
  • Promotion and High Price Identification: Highlights products with prices below the median (promotions) and above the median (high prices), aiding in the choice of economical alternatives.
  • Brand and Product Comparison: Presents brand options for the same product, indicating the most financially advantageous alternative.
  • Price Evolution: Traces a history of the price variation of diet items and the total cost, offering a longitudinal perspective on food economy.

Conclusion

The Economical Diet project represents a significant advancement in how individuals can monitor and manage the costs of their nutrition. By combining a sophisticated approach to data extraction and analysis with a powerful visualization interface, the project encourages informed decisions while promoting healthy and economical eating.

To use this project structure you will need to follow the steps below.

Requirements

To use this project properly, you will need to install: - python - git - pyenv - poetry

To install and configure Pyenv and Poetry in Windows, check this video. To learn how to manage multiple python versions using pyenv, check this article.

Poetry local environment config

To create the .venv local folder, type in terminal: *for this project only, add the argument --local at the end of the command. Below is the poetry global settings.

poetry config virtualenvs.create true
poetry config virtualenvs.in-project true

DuckDB install

Windows:

winget install DuckDB.cli

For other OS, check it: duckdb download page

Installation Steps

Git Clone

Open a terminal window (cmd, bash, or anything with git commands) and type:

git clone https://github.com/alanceloth/localGroceryPrices.git [NEW_PROJECT_NAME]
cd [NEW_PROJECT_NAME]
git init

Create new GitHub Repo from existing one

In the terminal window, type:

gh repo create

Choose the third option: Push an existing local repository to GitHub. Choose the current path (just put a dot "." and hit enter) Follow the instructions on screen, add a remote called 'master', and that's it!

Setting up the environment

We will need python 3.12.2, and to get this version we will use pyenv. In the same terminal window, type:

If you never used pyenv, or if you don't have the 3.12.2 version in your pyenv:

pyenv update
pyenv install --l

If you find the 3.12.2, then it's everything correct.

pyenv install 3.12.2

To check the python versions installed, use this:

pyenv versions

You will notice that one of the versions will have a * symbol. This indicates that the system is using this version. You can also check the default python version used by the system with this:

which python

If you have the 3.12.2 version in your pyenv: To use the project python version (3.12.2), use the command below:

pyenv local 3.12.2

Poetry

To initialize the poetry in the project, type in the terminal:

poetry env use 3.12.2
poetry shell
poetry install --no-root

Testing

In the terminal:

duckdb

If the above code don't work properly, go to the duckdb download page and get the latest zip file. Extract the duckdb.exe and put inside your project folder. This will allow you to use the duckdb in the terminal.
